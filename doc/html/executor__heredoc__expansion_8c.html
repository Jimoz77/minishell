static int can_expand_heredoc ( char * var_end)
static

Vérifie si l'expansion d'une variable est possible dans un heredoc.

Paramètres
var_endPointeur vers le caractère suivant le nom de variable
Renvoie
int 1 si l'expansion est possible, 0 sinon

Une variable peut être étendue si le caractère suivant n'est pas :

  • Une lettre (a-z, A-Z)
  • Un chiffre (0-9)
  • Un underscore (_) Cela délimite correctement la fin du nom de variable.

Étend une variable d'environnement dans un heredoc.

Paramètres
lineLigne originale contenant la variable
currentPointeur vers le '$' dans la ligne
var_endPointeur vers la fin du nom de variable
shellStructure shell contenant l'environnement
Renvoie
char* Nouvelle ligne avec la variable remplacée par sa valeur

Processus d'expansion :

  1. Extrait le nom de la variable
  2. Recherche sa valeur dans l'environnement
  3. Reconstruit la ligne avec la valeur (ou chaîne vide si non trouvée)
  4. Libère la ligne originale et retourne la nouvelle

Paramètres

currentPointeur vers le '$' dans la ligne
shellStructure shell contenant le code de sortie
Renvoie
char* Nouvelle ligne avec $? remplacé par sa valeur

Remplace $? par le code de sortie de la dernière commande :

  1. Convertit shell->exit_status en chaîne
  2. Reconstruit la ligne en remplaçant $? par sa valeur
  3. Libère la ligne originale et retourne la nouvelle

char * expand_heredoc_line ( char * line,
t_shell * shell )

Étend les variables d'environnement dans une ligne de heredoc.

Paramètres
lineLigne à traiter
shellStructure shell contenant l'environnement
Renvoie
char* Ligne avec variables étendues, ou ligne originale si pas de variables

Parcourt la ligne à la recherche de variables ($VAR) et les étend :

  • $? : remplacé par le code de sortie
  • $VAR : remplacé par la valeur de la variable d'environnement L'expansion se fait récursivement (une variable par appel). Retourne la ligne originale si aucune variable n'est trouvée.
