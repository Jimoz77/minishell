int apply_node_redirections ( t_node * node,
t_redirect * red )

Applique toutes les redirections d'un nœud et configure les descripteurs.

Paramètres
nodeNœud contenant les redirections à appliquer
redStructure de redirection à configurer
Renvoie
int 1 si succès, 0 si erreur

Processus complet d'application des redirections :

  1. Vérifie la validité du nœud et la présence de redirections
  2. Parcourt toutes les redirections de la liste chaînée
  3. Applique chaque redirection individuellement
  4. Configure les descripteurs standards (stdin/stdout) avec dup2()
  5. Gère les erreurs de dup2() et affiche les messages appropriés

Les redirections sont appliquées dans l'ordre d'apparition dans la liste. En cas d'erreur sur une redirection, l'application s'arrête immédiatement.

t_redirect * red,
t_node * node )
static

Applique une redirection spécifique selon son type.

Paramètres
redirStructure de redirection à appliquer
redStructure de redirection pour stocker les descripteurs
nodeNœud associé (utilisé pour les heredocs)
Renvoie
int 1 si succès, 0 si erreur

Dispatche vers la fonction appropriée selon le type de redirection :

  • TOKEN_REDIRECT_IN : apply_redirect_in() pour redirection d'entrée (<)
  • TOKEN_REDIRECT_OUT/TOKEN_APPEND : apply_redirect_out() pour sortie (>, >>)
  • TOKEN_HEREDOC : apply_heredoc_redir() pour here document (<<) Retourne 1 pour les types non reconnus (pas d'erreur).
