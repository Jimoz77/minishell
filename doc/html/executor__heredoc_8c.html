Paramètres

Traite tous les heredocs dans l'arbre syntaxique avec expansion.

Paramètres
nodeRacine de l'arbre à traiter

Parcourt récursivement tout l'arbre syntaxique et traite tous les heredocs avant l'exécution. Cette approche permet de traiter tous les heredocs en amont et évite les problèmes de synchronisation lors de l'exécution avec pipes et forks.

Traite une ligne de heredoc avec expansion.

Paramètres
delimiterDélimiteur de fin du heredoc
shellStructure shell pour l'expansion
contentPointeur vers le contenu accumulé
delim_lenLongueur du délimiteur
Renvoie
int 1 pour continuer, 0 pour terminer, -1 si interruption

Processus de traitement d'une ligne :

  1. Active le mode heredoc pour readline
  2. Lit une ligne avec le prompt "> "
  3. Vérifie les interruptions (Ctrl+C)
  4. Compare avec le délimiteur pour voir si on doit terminer
  5. Applique l'expansion des variables si nécessaire
  6. Ajoute la ligne au contenu

Traite tous les heredocs d'un nœud avec expansion.

Paramètres
nodeNœud contenant des heredocs à traiter
shellStructure shell pour l'expansion

Pour chaque heredoc du nœud :

  1. Vérifie qu'il n'est pas déjà traité
  2. Lit le contenu avec expansion
  3. Convertit le contenu en descripteur de fichier
  4. Marque le heredoc comme traité Gère les erreurs d'allocation et de conversion.

Lit tout le contenu d'un heredoc avec expansion.

Paramètres
delimiterDélimiteur de fin du heredoc
shellStructure shell pour l'expansion
Renvoie
char* Contenu complet du heredoc, NULL si erreur

Lit ligne par ligne jusqu'au délimiteur :

  1. Initialise le contenu vide
  2. Calcule la longueur du délimiteur
  3. Traite chaque ligne avec process_heredoc_line()
  4. S'arrête quand le délimiteur est rencontré ou en cas d'interruption Retourne tout le contenu accumulé ou NULL si interruption.
