Exécute une commande (builtin ou externe) dans le contexte d'un pipe.

Paramètres
nodeNœud de commande à exécuter
envpPointeur vers l'environnement
redStructure de redirection configurée
Renvoie
int Code de retour de l'exécution

Dispatche l'exécution selon le type de commande :

  • Builtin : exécute directement et nettoie les redirections
  • Externe : résout le chemin et exécute avec execve()

Pour les externes : cette fonction ne retourne qu'en cas d'erreur (execve ne retourne jamais si succès).

◆ handle_pipe_status()

int handle_pipe_status ( int status)

Traite le code de retour d'un processus dans un contexte de pipe.

Paramètres
statusCode de statut retourné par waitpid()
Renvoie
int Code de retour normalisé

Analyse le statut de sortie dans le contexte des pipes :

  • Si terminé par signal : retourne 128 + numéro du signal
  • Si SIGQUIT : affiche "Quit (core dumped)"
  • Si sortie normale : retourne le code de sortie
  • Par défaut : retourne 1

int prepare_cmd_tokens ( t_node * node,
t_shell * shell )

Prépare les tokens de commande pour l'exécution dans un pipe.

Paramètres
nodeNœud de commande à préparer
shellStructure shell
Renvoie
int 0 si succès, -1 si erreur

Processus de préparation optimisé pour les pipes :

  1. Sauvegarde les tokens originaux
  2. Crée de nouveaux tokens à partir de la commande
  3. Applique l'expansion et la suppression des guillemets
  4. Met à jour la commande avec les valeurs finales
  5. Restaure les tokens originaux et libère les temporaires

Traite l'expansion des variables et la suppression des guillemets.

Paramètres
tokensListe de tokens à traiter
shellStructure shell avec l'environnement

Processus en deux étapes :

  1. Expansion des variables : applique l'expansion sur les tokens non quotés
  2. Suppression des guillemets : construit les valeurs finales sans guillemets Préserve les tokens entre guillemets simples (pas d'expansion).
